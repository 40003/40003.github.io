MySQL知识点(关系型数据库)

什么叫关系型数据库:(Oracle.DB2.MySQL.SQL Server)

具有固定的列数和任意的行数

缺点

关系型数据库存在数据模型 性能 拓展伸缩性的缺点

NoSQL:非关系型数据库

数据库的概述(DataBase DB):

  1. 数据库是按照数据结构来组织 存储和管理数据的仓库(其实数据库就是磁盘中的文件+一个管理系统)

SQL(结构化查询语言):

是关系型数据库标准语言

特点:简单 灵活 功能强大

SQL包含6部分

  1. 数据查询语言(DQL)
    1. SELECT WHERE ORDER BY GROUP BY HAVING
  2. 数据操作语言(DML)
    1. INSERT UPDATE DELETE
  3. 事务处理语言(TPL)
  4. 数据控制语言(DCL)
  5. 数据定义语言(DDL)
    1. CREATE DROP

什么是表:

MySQL是一种关系型数据库 关系型数据库最重要的语言就是表

表具有固定的列数和任意的行数 在数学上称为关系

常用的操作命令:

  1. show databases;查看数据库
  2. user p2p;使用哪个数据库
  3. show tables;展示数据库中的表
  4. create databse(table) demo;创建demo数据库(表 )
  5. drop databse(table) demo;创建demo数据库(表 )
  6. DESC table xxx;查看表的定义
  7. show create table xxx;查看表的创建语句

MySQL的存储引擎(常用的InnoDB)

MySQL中数据用不同的技术存储在文件(或者内存)中,这些技术每一种技术使用不同的存储机制 索引技巧 锁定水平并且最终提供不同的功能和能力

通过选择不同的技术 可以获得额外的速度或者功能 从而改善你的应用的整体功能

MyISAM:拥有较高的插入 查询速度 但不支持事务 不支持外键

InnoDB:支持事务 支持外键 支持行级锁定 性能较低

MySQL常用的整数类型

  1. 整数:tinyint smallint mediumint int/integer bigint
  2. 小数:float double decimal
  3. 字符:char varchar tinytext text mediumtext longtext
    1. char 和 varchar的区别: char(5)不管char的长度是多少(“”,”1”)都占5个字节 varchar(5)是实际字节+1
  4. 日期类型:year date time datetime timestamp

表的约束

  1. 非空约束 not null 不允许某列的内容为空
  2. 设置列的默认值 default
  3. 唯一约束 unique 在该表中 该列的内容必须唯一
  4. 主键约束 primary key 非空且唯一
  5. 主键自增长 auto_increament 从1开始步长为1
  6. 外键约束 foreign key a表中的外键列的值必须参照b表中的某一列
